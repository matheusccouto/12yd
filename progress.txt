## Iteration 2: client.py buildId-strip + shared httpx (2026-07-09)

### Task
Apply the linchpin fix to `client.py` (PRD #1, #2, #7): strip FotMob's
buildId from cache keys so cached responses survive deployment rotations,
and replace per-request `httpx.Client` creation with a single lazy-init
shared instance.

### Key decisions
- `_cache_key(url)` strips `/_next/data/<buildId>/` → `/_next/data/_/`
  before computing the filesystem-safe key. Same logical resource → same
  key across buildIds.
- `FotMobClient._ensure_http()` lazy-inits a single `httpx.Client` and
  returns it on subsequent calls.
- Added `FotMobClient.close()` for explicit cleanup.

### Files changed
- `src/twelveyards/client.py` — buildId-strip in `_cache_key`, shared httpx
- `tests/test_client.py` — 7 new tests (5 buildId-strip + 2 shared-http)
